Project Summary

The goal of this project is to create a game where players are able to interact with their environment (pick up/run into objects), and each interaction would change the scene to a different shading style using stylized shaders. We will be combining different shaders to create a unique world depending on what the player selects.

Problem Description

In most games, the visual style remains consistent throughout, limiting how graphics can reflect changing gameplay or player interaction. Our project explores the challenge of dynamically switching between distinct shading styles — such as cel shading, photorealism, and retro graphics — in a single interactive scene. This involves combining different shaders in a cohesive way, managing performance, and creating smooth, visually engaging transitions. By allowing players to trigger these shader changes through environmental interactions, we aim to create a playful, immersive experience that pushes the boundaries of traditional game aesthetics.

To tackle this, we will build a Unity-based 1st person game that incorporates 3 different possible base shaders. Instead of writing shaders from scratch, we will modify and customize existing ones to suit our visual and interactive goals. The player will be able to interact with items or environments — like bumping into a "water" object to switch to a fluid simulation style, or picking up a pixelated item to enter a retro shader space. Furthermore, when the player interacts with multiple items together, we switch to a shader that visually looks like a combination of the base shaders. As a stretch goal, we plan to explore the implementation in VR to further enhance the immersion, using hand interactions to trigger these visual transformations.

Goals and Deliverables

  1. What we plan to deliver:
    • Baseline Goal:
      • Create a scene in Unity with 3 different possible base shaders. We will modify existing shaders rather than using them from scratch.
      • We will also write custom shaders that are combinations of the base shaders (3 combos that are pairs, 1 combo consisting of all 3 base shaders).
      • Our current idea is to hard-code each of these combination shaders in High-Level Shader Language (HLSL) based on an understanding of the mechanics of their underlying shaders.
  2. What we hope to deliver:
    • Stretch Goals:
      • Create an interactive VR game in Unity where players are capable of changing the world's shading style by interacting with different objects (representing different shaders) in the environment.
      • Allow single objects in the VR world to be shaded differently from the rest of the world.
      • Create a generalized algorithm for combining shaders - create a toggle (or multiple toggles) that changes the influence of each shader on the final result (i.e. 70% shader 1, 30% shader 2).
  3. Performance metrics:
    • Smooth and quick transition when shading style changes
    • Intuitive interactions between the player and shader objects
    • The combined shaders should be visually convincing and appealing
    • Have objects of different textures in the scene and be able to render them with different shaders appropriately
    • FPS >= 24
  4. What questions do you plan to answer?
    • How can we efficiently manage and switch between multiple shaders on shared scene geometry or objects without causing noticeable rendering artifacts or performance issues in Unity?
    • How can we blend or layer multiple shader effects on a single object or scene in Unity without breaking visual consistency?
    • What are effective ways to design interactions that trigger shader transitions in a way that feels intuitive and immersive for the player?

  • Finalize the shader styles that we want to include.
    • Pool in shader styles we are interested in and vote
    • Decide on whether we want to write a shader from scratch and what it would look like
    • Create concept sketches on the expected/idealized results

  • Implement and test all base shaders in the scene

    • Basic in-game trigger for switching shaders
  • Implement combo shaders

Week 3:

  • Finish testing combo shaders
  • Develop player interactions (e.g., object collision, pickup) that trigger shader changes

  • Finalize all shaders and interactions

  • Check against performance metrics and make sure we have good render speed and visual consistency

  • Prepare final demo, presentation, and documentation
